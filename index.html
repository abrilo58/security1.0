5.1 The Troubleshooting Methodology

Question 1

Which of the following steps of the troubleshooting methodology would most likely include checking through each level of the OSI model after the problem has been identified?

A. Establish a theory.

B. Implement the solution.

C. Create a plan of action.

D. Verify functionality.

Show Answer & Explanation ▶

Correct Answer: A. Establish a theory

Why this is correct: The "Establish a theory" step involves investigating potential causes of the problem, which often includes examining each layer of the OSI model to identify where the issue might be occurring.

Why others are incorrect:

  • Implement the solution: This step comes after identifying the cause and creating a plan.
  • Create a plan of action: This involves developing steps to resolve the issue, not diagnosing it.
  • Verify functionality: This is the final step to confirm the problem is resolved.

Question 2

Which of the following is the next step to take after successfully testing a root cause theory?

A. Determine resolution steps.

B. Duplicate the problem in a lab.

C. Present the theory for approval.

D. Implement the solution to the problem.

Show Answer & Explanation ▶

Correct Answer: A. Determine resolution steps

Why this is correct: After confirming a theory through testing, the next logical step is to determine the specific steps needed to resolve the issue (resolution steps) before implementing them.

Why others are incorrect:

  • Duplicate the problem in a lab: This would occur before testing the theory, not after.
  • Present the theory for approval: Approval isn't typically part of standard troubleshooting methodology.
  • Implement the solution: This comes after determining resolution steps.

Correct Answer: B. Establish a plan of action

Why this is correct: After testing and confirming a theory (in this case about a faulty NIC), the next step is to establish a plan of action for resolving the issue.

Why others are incorrect:

  • Develop a theory: This occurs before testing the theory.
  • Implement the solution: This comes after creating a plan.
  • Document the findings: This is the final step after verifying the solution.

D. Determine any changes.

Why others are incorrect:

  • Identify the symptoms: This is part of the initial problem identification.
  • Gather information: This occurs before forming theories.
  • Determine any changes: This is part of initial investigation but doesn't confirm theories.

5.2 Common Cabling and Physical Interface Issues

Question 14

A support agent receives a report that a remote user's wired devices are constantly disconnecting and have slow speeds. Upon inspection, the support agent sees that the user's coaxial modem has a signal power of ~97dB. Which of the following should the support agent recommend to troubleshoot the issue?

A. Removing any splitters connected to the line

B. Switching the devices to wireless

C. Moving the devices closer to the modem

D. Lowering the network speed

Show Answer & Explanation ▶

Correct Answer: A. Removing any splitters connected to the line

Why this is correct: A signal power of ~97dB is extremely high (ideal range is typically -15dB to +15dB). Splitters weaken signal strength, so removing them would reduce the excessive signal power causing disconnections and slow speeds.

Why others are incorrect:

  • Switching to wireless: Doesn't address the underlying coaxial signal issue.
  • Moving devices closer: Irrelevant for wired connections and signal power issues.
  • Lowering network speed: Doesn't fix the physical signal problem.

Question 15

A customer is adding fiber connectivity between adjacent buildings. A technician terminates the multimode cable to the fiber patch panel. After the technician connects the fiber patch cable, the indicator light does not turn on. Which of the following should a technician try first to troubleshoot this issue?

A. Reverse the fibers.

B. Reterminate the fibers.

C. Verify the fiber size.

D. Examine the cable runs for visual faults.

Show Answer & Explanation ▶

Correct Answer: A. Reverse the fibers

Why this is correct: Fiber connections require proper Tx/Rx alignment. The simplest first check is ensuring transmit and receive fibers aren't reversed at either end.

Why others are incorrect:

  • Reterminate: Should only be done after simpler checks like fiber reversal.
  • Verify fiber size: Less likely to cause complete link failure than reversed fibers.
  • Examine for faults: More time-consuming than checking fiber polarity first.

Question 16

After running a Cat 8 cable using passthrough plugs, an electrician notices that connected cables are experiencing a lot of crosstalk. Which of the following troubleshooting steps should the electrician take first?

A. Inspect the connectors for any wires that are touching or exposed.

B. Restore default settings on the connected devices.

C. Terminate the connections again.

D. Check for radio frequency interference in the area.

Show Answer & Explanation ▶

Correct Answer: A. Inspect the connectors for any wires that are touching or exposed

Why this is correct: Crosstalk is typically caused by improper termination where wire pairs are untwisted too much or make contact with other pairs. Passthrough plugs are particularly prone to this if wires aren't properly trimmed.

Why others are incorrect:

  • Restore default settings: Crosstalk is a physical layer issue, not configuration.
  • Terminate again: Should only be done after inspecting the current termination.
  • Check RF interference: RFI causes different symptoms than crosstalk.

Question 17

A network administrator is notified that a user cannot access resources on the network. The network administrator checks the physical connections to the workstation labeled User 3 and sees the Ethernet is properly connected. However, the network interface's indicator lights are not blinking on either the computer or the switch. Which of the following is the most likely cause?

A. The switch failed.

B. The default gateway is wrong.

C. The port is shut down.

D. The VLAN assignment is incorrect.

Show Answer & Explanation ▶

Correct Answer: C. The port is shut down

Why this is correct: No link lights on either device strongly suggests a physical layer issue. An administratively shut down port would explain this behavior while other options wouldn't affect the physical link lights.

Why others are incorrect:

  • Switch failure: Other users would likely be affected.
  • Wrong gateway: Wouldn't prevent link lights from activating.
  • VLAN assignment: Link lights would still be on if VLAN was misconfigured.

Question 20

A network administrator determines that some switch ports have more errors present than expected. The administrator traces the cabling associated with these ports. Which of the following would most likely be causing the errors?

A. Shielded conduit

B. Plenum spaces

C. Voltage transformers

D. Hydraulic lifts

Show Answer & Explanation ▶

Correct Answer: C. Voltage transformers

Why this is correct: Voltage transformers can create electromagnetic interference (EMI) that induces errors in nearby network cables, especially if they're unshielded or run parallel to power lines.

Why others are incorrect:

  • Shielded conduit: Actually protects against EMI rather than causing it.
  • Plenum spaces: Relates to fire safety, not error generation.
  • Hydraulic lifts: Unlikely to cause consistent network errors.

5.3 Common Issues with Network Services

Question 26

A user is unable to navigate to a website because the provided URL is not resolving to the correct IP address. Other users are able to navigate to the intended website without issue. Which of the following is most likely causing this issue?

A. Hosts file

B. Self-signed certificate

C. Nameserver record

D. IP helper

Show Answer & Explanation ▶

Correct Answer: A. Hosts file

Why this is correct: The hosts file can override DNS resolution on a single machine. If it contains an incorrect entry for the URL, it would explain why only this user has the issue while others can access the site normally.

Why others are incorrect:

  • Self-signed certificate: Would cause SSL warnings, not resolution failures.
  • Nameserver record: Would affect all users if DNS was misconfigured.
  • IP helper: Relates to DHCP relay, not DNS resolution.

Question 27

A user reports having intermittent connectivity issues to the company network. The network configuration for the user reveals the following: IP address: 192.168.1.10 - Subnet mask: 255.255.255.0 - Default gateway: 192.168.1.254 - The network switch shows an ARP table with the same MAC for two different IPs. Which of the following is the most likely cause of the user's connection issues?

A. A port with incorrect VLAN assigned

B. A switch with spanning tree conflict

C. Another PC with manually configured IP

D. A router with overlapping route tables

Show Answer & Explanation ▶

Correct Answer: C. Another PC with manually configured IP

Why this is correct: The same MAC appearing for two IPs indicates an IP conflict, most commonly caused by another device manually configured with the same IP (192.168.1.10) as the reporting user's device.

Why others are incorrect:

  • Incorrect VLAN: Would prevent all connectivity, not cause intermittent issues.
  • Spanning tree: Would cause broader network outages, not single-user issues.
  • Overlapping routes: Would affect traffic routing beyond the local subnet.

Question 35

A network technician is troubleshooting network latency and has determined the issue to be occurring between two network switches (Switch10 and Switch11). 5.5 Appropriate Tool or Protocol

Question 51

  • VLAN: Virtual LANs segment traffic but don't identify physical ports.

Why this is correct: Traceroute (tracert) shows the complete path traffic takes to a destination, allowing the technician to see which internet link is being used based on the first hop.

  • nslookup: Used for DNS queries, not path determination.
  • ping: Tests connectivity but doesn't show the route.
