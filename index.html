1.0 General Security Concepts

Question 1

Which of the following is used to add extra complexity before using a one-way data transformation algorithm?

A. Key stretching

B. Data masking

C. Steganography

D. Salting

Show Answer & Explanation ▶

Correct Answer: D. Salting

Why this is correct: Salting adds random data (a "salt") to input before hashing (a one-way transformation). This prevents attackers from using precomputed tables (rainbow tables) to reverse hashes, as each salt creates a unique hash even for identical inputs.

Why others are incorrect:

  • Key stretching: Slows down brute-force attacks by making hashing computationally intensive (e.g., PBKDF2), but doesn’t add randomness like salting.
  • Data masking: Hides sensitive data (e.g., showing only last 4 digits of a credit card), but doesn’t modify input for hashing.
  • Steganography: Hides data within other files (e.g., images), unrelated to hashing complexity.

Question 2

A security administrator would like to protect data on employees’ laptops. Which of the following encryption techniques should the security administrator use?

A. Partition

B. Asymmetric

C. Full disk

D. Database

Show Answer & Explanation ▶

Correct Answer: C. Full disk

Why this is correct: Full disk encryption (FDE) encrypts all data on a laptop’s storage, including the OS and files. If the laptop is stolen, the data remains unreadable without the decryption key (e.g., BitLocker, FileVault).

Why others are incorrect:

  • Partition encryption: Only protects specific partitions, leaving other data (e.g., system files) vulnerable.
  • Asymmetric encryption: Uses public/private keys (e.g., RSA) but is slower and typically used for small data (e.g., emails), not entire disks.
  • Database encryption: Protects only database files, not the entire laptop.

Question 3

Which of the following security control types does an acceptable use policy best represent?

A. Detective

B. Compensating

C. Corrective

D. Preventive

Show Answer & Explanation ▶

Correct Answer: D. Preventive

Why this is correct: An acceptable use policy (AUP) is a preventive control. It outlines rules for using company resources (e.g., no illegal downloads) to prevent misuse before it happens, reducing risks like malware infections or data breaches.

Why others are incorrect:

  • Detective: Identifies incidents after they occur (e.g., log monitoring).
  • Compensating: Alternative controls when primary ones fail (e.g., manual reviews if automation breaks).
  • Corrective: Fixes issues post-incident (e.g., patching after a hack).

Question 4

An IT manager informs the entire help desk staff that only the IT manager and the help desk lead will have access to the administrator console of the help desk software. Which of the following security techniques is the IT manager setting up?

A. Hardening

B. Employee monitoring

C. Configuration enforcement

D. Least privilege

Show Answer & Explanation ▶

Correct Answer: D. Least privilege

Why this is correct: The principle of least privilege (PoLP) grants users the minimum access needed to perform their jobs. Here, only two roles get admin console access, reducing risk if other accounts are compromised.

Why others are incorrect:

  • Hardening: Secures systems by removing vulnerabilities (e.g., disabling unused services), not access control.
  • Employee monitoring: Tracks user activity (e.g., screen recording), unrelated to access restrictions.
  • Configuration enforcement: Ensures systems comply with policies (e.g., auto-updates), not access levels.

Question 5

Which of the following allows for the attribution of messages to individuals?

A. Adaptive identity

B. Non-repudiation

C. Authentication

D. Access logs

Show Answer & Explanation ▶

Correct Answer: B. Non-repudiation

Why this is correct: Non-repudiation ensures a sender cannot deny sending a message. Techniques like digital signatures (e.g., S/MIME) bind the message to the sender’s identity, providing legal proof of origin.

Why others are incorrect:

  • Adaptive identity: Adjusts access based on behavior (e.g., login location), but doesn’t prove message ownership.
  • Authentication: Verifies identity (e.g., passwords), but doesn’t prevent denial of actions.
  • Access logs: Record who accessed resources, but lack cryptographic proof like non-repudiation.

Question 6

Which of the following has been implemented when a host-based firewall on a legacy Linux system allows connections from only specific internal IP addresses?

A. Compensating control

B. Network segmentation

C. Transfer of risk

D. SNMP traps

Show Answer & Explanation ▶

Correct Answer: A. Compensating control

Why this is correct: A compensating control is a secondary measure implemented when primary security controls aren't feasible. Here, the host-based firewall (compensating control) restricts access because the legacy system can't be updated or segmented properly.

Why others are incorrect:

  • Network segmentation: Divides networks into subnetworks (e.g., VLANs), but this describes a host-based rule, not network-level changes.
  • Transfer of risk: Shifts risk to a third party (e.g., insurance), unrelated to firewall rules.
  • SNMP traps: Alerts for network events, not access control.

Question 7

A. Corrective

B. Preventive

C. Detective

D. Deterrent

Show Answer & Explanation ▶

Correct Answer: C. Detective

Why this is correct: A SIEM (Security Information and Event Management) system is a detective control. It identifies and alerts on suspicious activity (e.g., failed logins) after it occurs, and manual log reviews further enhance detection.

Why others are incorrect:

  • Corrective: Fixes issues post-incident (e.g., patching), but SIEM doesn’t automate fixes.
  • Preventive: Blocks incidents (e.g., firewalls), but SIEM doesn’t stop attacks.
  • Deterrent: Discourages attacks (e.g., warning banners), but SIEM doesn’t deter, it detects.

Question 8

Security controls in a data center are being reviewed to ensure data is properly protected and that human life considerations are included. Which of the following best describes how the controls should be set up?

A. Remote access points should fail closed.

B. Logging controls should fail open.

C. Safety controls should fail open.

D. Logical security controls should fail closed.

Show Answer & Explanation ▶

Correct Answer: C. Safety controls should fail open.

Why this is correct: Safety controls (e.g., fire doors) must fail open to prioritize human life. If a system fails, doors unlock to allow evacuation, even if it compromises data security temporarily.

Why others are incorrect:

  • Remote access/Logical controls: Should fail closed (deny access) to prevent unauthorized entry during failures.
  • Logging controls: Should neither fail open nor closed; logs should be preserved but not impact safety.

Question 9

Which of the following is used to validate a certificate when it is presented to a user?

A. OCSP

B. CSR

C. CA

D. CRC

Show Answer & Explanation ▶

Correct Answer: A. OCSP

Why this is correct: OCSP (Online Certificate Status Protocol) checks if a certificate is valid or revoked in real-time when presented. For example, your browser uses OCSP to verify a website’s SSL certificate before establishing a secure connection.

Why others are incorrect:

  • CSR (Certificate Signing Request): A request to a CA to issue a certificate, not for validation.
  • CA (Certificate Authority): Issues certificates but doesn’t validate them in real-time.
  • CRC (Cyclic Redundancy Check): Detects data errors (e.g., in files), unrelated to certificates.

Question 10

A. Data in use

B. Data in transit

C. Geographic restrictions

D. Data sovereignty

Show Answer & Explanation ▶

Correct Answer: B. Data in transit

Why this is correct: A VPN (Virtual Private Network) encrypts data in transit between locations, preventing eavesdropping. For example, it secures emails sent from the branch to headquarters over the internet.

Why others are incorrect:

  • Data in use: Protected by memory encryption (e.g., Intel SGX), not VPNs.
  • Geographic restrictions: Bypassed by VPNs (e.g., accessing region-locked content), but not their primary purpose.
  • Data sovereignty: Laws about data storage locations, unrelated to VPN encryption.

Question 12

Which of the following security concepts is the best reason for permissions on a human resources fileshare to follow the principle of least privilege?

A. Integrity

B. Availability

C. Confidentiality

D. Non-repudiation

Show Answer & Explanation ▶

Correct Answer: C. Confidentiality

Why this is correct: Confidentiality ensures sensitive data (e.g., HR records) is only accessible to authorized personnel. Least privilege restricts access to "need-to-know" roles, preventing unauthorized viewing or leaks.

Why others are incorrect:

  • Integrity: Ensures data isn’t altered, but least privilege primarily prevents access, not modification.
  • Availability: Ensures data is accessible, but least privilege may limit availability (e.g., denying access to some users).
  • Non-repudiation: Proves actions (e.g., who modified a file), unrelated to access permissions.

Question 13

A. Compensating

B. Detective

C. Preventive

D. Corrective

Correct Answer: B. Detective

Why this is correct: Detective controls identify incidents after they occur. Log reviews help trace how the ransomware entered the system (e.g., via a phishing email) and its impact, enabling future prevention.

Why others are incorrect:

Question 14

A company must ensure sensitive data at rest is rendered unreadable. Which of the following will the company most likely use?

A. Hashing

B. Tokenization

C. Encryption

D. Segmentation

Show Answer & Explanation ▶

Correct Answer: C. Encryption

Why this is correct: Encryption transforms data (e.g., files, databases) into unreadable ciphertext using a key. Only authorized users with the key can decrypt it, protecting data at rest (e.g., on hard drives).

Why others are incorrect:

  • Hashing: One-way transformation (e.g., for passwords), but hashed data cannot be reversed to its original form.
  • Tokenization: Replaces data with tokens (e.g., credit card numbers with random IDs), but original data is stored elsewhere.
  • Segmentation: Isolates networks, but doesn’t render data unreadable.

Question 15

A. Encryption at rest

B. Masking

C. Data classification

D. Permission restrictions

Show Answer & Explanation ▶

Correct Answer: A. Encryption at rest

Why this is correct: Encryption at rest (e.g., BitLocker) ensures data on laptops remains unreadable if stolen. Even if thieves access the hard drive, they can’t decrypt files without the key.

Why others are incorrect:

  • Masking: Hides parts of data (e.g., showing only XXX-XXX-1234 for credit cards), but doesn’t protect full files.
  • Data classification: Labels data sensitivity (e.g., "Confidential"), but doesn’t secure it.
  • Permission restrictions: Limits access but doesn’t encrypt stored data.

Question 16

Why this is correct: This option covers all three MFA factors: - Something you know: Password - Something you have: Authentication token (e.g., RSA SecurID) - Something you are: Thumbprint (biometrics).

Question 17

Which of the following describes the process of concealing code or text inside a graphical image?

A. Symmetric encryption

B. Hashing

C. Data masking

D. Steganography

Show Answer & Explanation ▶

Correct Answer: D. Steganography

Why this is correct: Steganography hides data (e.g., text, code) within other files (e.g., images, audio). Unlike encryption, it doesn’t scramble data but makes it invisible to casual inspection (e.g., altering pixel colors subtly).

Why others are incorrect:

  • Symmetric encryption: Scrambles data with a key but doesn’t hide it in another file.
  • Hashing: Creates a fixed-size checksum (e.g., SHA-256), irreversible for hiding data.
  • Data masking: Obscures parts of data (e.g., credit card numbers), not concealment in files.

Question 20

In order to strengthen a password and prevent a hacker from cracking it, a random string of 36 characters was added to the password. Which of the following best describes this technique?

A. Key stretching

B. Tokenization

C. Data masking

D. Salting

Show Answer & Explanation ▶

Correct Answer: D. Salting

Why this is correct: Salting adds random data (the "salt") to passwords before hashing. This ensures identical passwords (e.g., "Password123") produce unique hashes, thwarting rainbow table attacks. The 36-character string is the salt.

Why others are incorrect:

  • Key stretching: Slows hashing (e.g., PBKDF2) but doesn’t add randomness.
  • Tokenization: Replaces sensitive data with tokens (e.g., payment processing), unrelated to passwords.
  • Data masking: Hides parts of data (e.g., SSNs), not password strengthening.

Question 21

A systems administrator uses a key to encrypt a message being sent to a peer in a different branch office. The peer then uses the same key to decrypt the message. Which of the following describes this example?

A. Symmetric

B. Asymmetric

C. Hashing

D. Salting

Show Answer & Explanation ▶

Correct Answer: A. Symmetric

Why this is correct: Symmetric encryption uses the same key to encrypt and decrypt data (e.g., AES). It's fast and efficient for bulk data encryption between trusted parties who can securely share the key.

Why others are incorrect:

  • Asymmetric: Uses a public/private key pair (e.g., RSA), where different keys encrypt/decrypt.
  • Hashing: One-way transformation (e.g., SHA-256) that cannot decrypt data.
  • Salting: Adds randomness to passwords before hashing, unrelated to message encryption.

Correct Answer: A. Residual

Question 33

A. Something you exhibit

B. Something you have

C. Somewhere you are

D. Someone you know

Show Answer & Explanation ▶

Correct Answer: B. Something you have

Why this is correct: MFA (Multi-Factor Authentication) requires multiple independent factors. "Something you have" (e.g., smartphone, hardware token) is one of the three core factors, along with "something you know" (password) and "something you are" (biometrics).

Question 34

Which of the following most likely describes why a security engineer would configure all outbound emails to use S/MIME digital signatures?

A. To meet compliance standards

B. To increase delivery rates

C. To block phishing attacks

D. To ensure non-repudiation

Show Answer & Explanation ▶

Correct Answer: D. To ensure non-repudiation

Why this is correct: S/MIME digital signatures cryptographically bind the sender's identity to the email. This prevents senders from later denying ("repudiating") they sent the message, as the signature is mathematically verifiable.

Why others are incorrect:

  • Compliance (A): While some regulations require non-repudiation, this is the result of the signature, not its primary purpose.
  • Delivery rates (B): Signatures don't affect email routing.
  • Phishing (C): S/MIME prevents email spoofing but doesn't block phishing (e.g., legitimate-looking malicious emails).

Real-world use: Legal or financial emails often use S/MIME to prove authenticity.

Question 44

Which of the following describes the difference between encryption and hashing?

A. Encryption protects data in transit, while hashing protects data at rest.

B. Encryption replaces cleartext with ciphertext, while hashing calculates a checksum.

C. Encryption ensures data integrity, while hashing ensures data confidentiality.

D. Encryption uses a public-key exchange, while hashing uses a private key.

Show Answer & Explanation ▶

Correct Answer: B. Encryption replaces cleartext with ciphertext, while hashing calculates a checksum.

Why this is correct:

  • Encryption: Reversible process that converts readable data (cleartext) to scrambled data (ciphertext) using a key. Example: AES encrypting a file.
  • Hashing: Irreversible one-way function that generates a fixed-size string (checksum) from data. Example: SHA-256 hash of a password.

Why others are incorrect:

  • A: Both can protect data in transit/at rest (usage depends on context).
  • C: Hashing ensures integrity (detects changes via checksum mismatch), encryption ensures confidentiality.
  • D: Hashing doesn't use keys; encryption can be symmetric (single key) or asymmetric (public/private).

Analogy: Encryption is like a locked safe (reversible), hashing is like a fingerprint (unique but can't reconstruct original).
